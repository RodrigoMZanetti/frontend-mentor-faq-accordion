## Project Overview

This project is a FAQ accordion component built as part of a front-end challenge. The goal was to create an interactive and accessible UI element using HTML, CSS, and JavaScript, focusing on clean structure, responsive design, and dynamic behavior.

## Features

- Accordion functionality (expand/collapse items)
- State management using JavaScript
- Accessible attributes (aria-expanded, aria-controls)
- Smooth animation using CSS
- Responsive design for mobile and desktop
- Icon toggle (+ / -) based on state

## Technologies Used

- HTML5 – semantic structure
- CSS3 – layout, animations, responsiveness
- JavaScript (Vanilla JS) – interactivity and state control

## Project Structure

- index.html – main structure
- style.css / blocks/ – styling and layout organization
- index.js – accordion logic and interaction
- assets/ – images and fonts

## How to Run the Project

- Clone or download the repository

✅ Completed as part of a front-end challenge

## Problem-Solving Approach

- Structuring accordion items as reusable components
- Using event listeners to control open/close behavior
- Synchronizing UI state with accessibility attributes
- Managing DOM traversal (parent → child relationships)
- Applying CSS grid for smooth expand/collapse animation

## What I Learned

- How to build an accordion component from scratch
- DOM traversal using closest and querySelector
- Synchronizing UI and accessibility (aria attributes)
- Responsive design adjustments
- Clean CSS structure and layout refinement
